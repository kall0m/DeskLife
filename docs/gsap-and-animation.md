# GSAP и анимациите в DeskLife

Този документ обяснява какво е GSAP, защо се използва и как е приложен в проекта.

## Какво е GSAP

GSAP означава **GreenSock Animation Platform**.

Това е JavaScript библиотека за създаване на плавни и контролирани анимации в уебсайтове и приложения.

GSAP може да анимира:

- позиция;
- прозрачност;
- размер;
- завъртане;
- цветове;
- SVG елементи;
- последователности от анимации;
- елементи според позицията на скрола.

## Защо използваме библиотека за анимации

Някои прости анимации могат да се направят само с CSS. Например hover състояние на бутон или плавна промяна на цвят.

GSAP е полезен, когато искаме:

- по-точен контрол върху началото и края;
- добра производителност;
- по-сложна последователност;
- анимация, свързана със скрол;
- лесно почистване на анимациите в React;
- еднакво поведение в различни браузъри.

## Как използваме GSAP в DeskLife

DeskLife използва GSAP за появата на заглавията при скрол.

Компонентът е:

```text
app/components/ScrollHeadingAnimations.tsx
```

Той намира заглавията в основното съдържание:

```tsx
const headings = gsap.utils.toArray<HTMLElement>("main h1, main h2, main h3");
```

След това всяко заглавие започва:

- малко по-надолу;
- невидимо.

И се анимира до:

- нормалната си позиция;
- пълна видимост.

```tsx
gsap.fromTo(
  heading,
  { autoAlpha: 0, y: 24 },
  {
    autoAlpha: 1,
    y: 0,
    duration: 0.7,
    ease: "power2.out",
  },
);
```

## Какво означават тези стойности

### `autoAlpha`

`autoAlpha` управлява едновременно прозрачността и видимостта на елемента.

```text
autoAlpha: 0 → елементът е невидим
autoAlpha: 1 → елементът е видим
```

### `y`

`y` премества елемента вертикално.

```text
y: 24 → започва 24 пиксела по-надолу
y: 0  → завършва на естественото си място
```

### `duration`

`duration: 0.7` означава, че анимацията продължава 0.7 секунди.

### `ease`

`ease: "power2.out"` определя как се променя скоростта.

Анимацията започва по-бързо и завършва по-меко. Това обикновено изглежда по-естествено от напълно линейно движение.

## Какво е ScrollTrigger

`ScrollTrigger` е GSAP plugin, който стартира анимации според позицията на скрола.

В проекта той се регистрира така:

```tsx
gsap.registerPlugin(ScrollTrigger);
```

За всяко заглавие използваме:

```tsx
scrollTrigger: {
  trigger: heading,
  start: "top 88%",
  once: true,
}
```

Това означава:

- `trigger: heading` — самото заглавие активира анимацията;
- `start: "top 88%"` — анимацията започва, когато горната част на заглавието достигне около 88% от височината на екрана;
- `once: true` — анимацията се изпълнява само веднъж.

## Защо анимацията се изпълнява отново при смяна на страница

Компонентът използва `usePathname()` от Next.js.

```tsx
const pathname = usePathname();
```

Effect-ът зависи от текущия маршрут:

```tsx
useEffect(() => {
  // създаване на анимациите
}, [pathname]);
```

Когато потребителят отвори друга страница, маршрутът се променя и компонентът създава правилните анимации за новото съдържание.

## Какво е `gsap.context()`

`gsap.context()` групира създадените анимации.

```tsx
const context = gsap.context(() => {
  // анимации
});
```

При премахване или обновяване на компонента извикваме:

```tsx
return () => context.revert();
```

Това почиства старите анимации и ScrollTrigger обекти. Така избягваме дублирани listeners и неочаквано поведение.

## Достъпност и `prefers-reduced-motion`

Някои потребители предпочитат по-малко движение поради чувствителност към анимации, проблеми с равновесието или лично предпочитание.

DeskLife проверява системната настройка:

```tsx
if (window.matchMedia("(prefers-reduced-motion: reduce)").matches) return;
```

Когато тази настройка е включена, GSAP анимациите за заглавията не се стартират.

Това е важен принцип: анимацията трябва да подобрява изживяването, а не да пречи на достъпността.

## Кога да използваме CSS и кога GSAP

### CSS е подходящ за:

- hover състояния;
- прости transitions;
- промяна на цвят;
- кратка анимация на един елемент;
- hamburger икона;
- показване и скриване на меню.

### GSAP е подходящ за:

- scroll анимации;
- сложни последователности;
- координирани анимации на много елементи;
- прецизен timing;
- по-сложни SVG анимации;
- анимации, които трябва да се управляват с JavaScript.

В DeskLife използваме и двата подхода. Това е по-добре от използването на GSAP за всяко малко движение.

## Добри принципи за анимация

- Анимацията трябва да има причина.
- Не трябва да забавя потребителя.
- Не трябва да пречи на четенето.
- Движението трябва да е последователно в целия интерфейс.
- Продължителността обикновено трябва да е кратка.
- Трябва да се уважава `prefers-reduced-motion`.
- Анимацията не трябва да скрива важна информация за твърде дълго.

## Обобщение

```text

      ↓
ScrollTrigger

      ↓

      ↓
Потребителят вижда
плавно появяване на заглавията
```

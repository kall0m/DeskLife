# GitHub и история на версиите

Този документ обяснява какво са Git, GitHub, commit-ите и защо историята на проекта е важна.

## Какво е Git

Git е система за проследяване на промени в кода.

Всеки път, когато направим важна промяна, можем да я запишем като отделна версия. Тази записана промяна се нарича `commit`.

```text
Промяна във файловете
        ↓
Git проверява какво е променено
        ↓
Създаваме commit
        ↓
Получаваме нова точка в историята
```

Git работи локално на компютъра на разработчика. Това означава, че историята може да съществува дори преди кодът да бъде качен онлайн.

## Какво е GitHub

GitHub е онлайн платформа, която съхранява Git repository-то на проекта.

Repository може да се мисли като папката на проекта заедно с цялата история на промените.

GitHub пази:

- файловете на проекта;
- commit-ите;
- различните версии на кода;
- информация кога е направена промяна;
- информация кой е направил промяната;
- branches, pull requests и issues, когато проектът използва такива.

GitHub не е самият публичен сайт. Той е мястото, където живеят кодът и неговата история.

## Как работи GitHub за DeskLife

При DeskLife процесът е приблизително следният:

```text
Променяме кода
      ↓
Проверяваме промените
      ↓
Създаваме commit
      ↓
Качваме commit-а в GitHub
      ↓
Vercel открива новата версия
      ↓
Изгражда и публикува сайта
```

Командата `push` изпраща локалните commit-и към GitHub.

## Какво е commit

Commit е записана стъпка в историята на проекта.

Един добър commit обикновено:

- съдържа свързани промени;
- има кратко и ясно описание;
- може да бъде разбран по-късно;
- не смесва много несвързани задачи.

Примерни commit съобщения:

```text
Add mobile navigation
Fix dark theme footer
Add BMI gauge to TDEE calculator
Document project technologies
```

Commit съобщението отговаря на въпроса: „Какво променя тази версия?“

## Защо историята на версиите е важна

### 1. Можем да видим как проектът се е развивал

Историята показва последователността на решенията и промените.

```text
Първа версия
    ↓
Добавена навигация
    ↓
Добавени форми
    ↓
Добавен dark mode
    ↓
Добавени анимации
```

### 2. Можем да открием кога се е появил проблем

Когато нещо спре да работи, commit историята помага да сравним последните версии и да намерим промяната, която е причинила проблема.

### 3. Можем да върнем предишна версия

Ако нова промяна не работи добре, Git позволява да възстановим по-стабилно състояние на проекта.

Това не означава, че трябва веднага да изтриваме новата работа. Обикновено първо проверяваме разликите и избираме най-безопасния начин за корекция.

### 4. Работата става по-разбираема

Когато commit-ите са малки и ясно описани, друг разработчик може по-лесно да разбере:

- какво е направено;
- защо е направено;
- кои файлове са засегнати;
- как се е променило поведението.

### 5. Историята е форма на документация

Кодът показва как работи проектът в момента. Историята показва как сме стигнали дотук.

Това е особено ценно при дългосрочни проекти, където решенията се натрупват с времето.

## Branches

Branch е отделна линия на работа.

```text
main
 ├── feature/mobile-menu
 └── feature/tdee-calculator
```

Главният branch често се казва `main`. Той обикновено съдържа стабилната версия на проекта.

За по-големи промени може да се създаде отделен branch, така че новата работа да не засяга веднага стабилната версия.

## Pull requests

Pull request е предложение даден branch да бъде обединен с друг branch, обикновено с `main`.

Pull request позволява:

- преглед на промените;
- коментари;
- автоматични проверки;
- обсъждане преди обединяване.

В малък личен проект може да се работи директно в `main`, но при екипна работа pull requests са много полезни.

## Добри навици

- Прави commit след завършена логическа стъпка.
- Използвай ясно commit съобщение.
- Не включвай пароли, API ключове или чувствителни данни.
- Проверявай сайта и build-а преди важен push.
- Не променяй историята без причина, особено когато други хора използват същия branch.
- Използвай историята, за да разбираш проекта, а не само като място за архив.

```text
Commit

GitHub

        ↓

```
